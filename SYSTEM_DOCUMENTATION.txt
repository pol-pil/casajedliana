CASA JEDLIANA SYSTEM DOCUMENTATION
Generated from the current Laravel + Inertia codebase inventory

1. SYSTEM SUMMARY

Casa Jedliana is a hotel/front desk management system built with:
- Laravel 12 backend
- Inertia.js v2
- React 19 + TypeScript frontend
- Tailwind CSS v4 UI styling
- Fortify authentication
- Wayfinder-generated frontend route/actions helpers
- Pest testing
- DOMPDF for Statement of Account printing

Primary business domains found in the codebase:
- rooms/accommodations
- bookings and guest records
- rates, charges, and booking types
- payments and balances
- operational dashboard
- reports and booking history
- admin user management
- hotel branding / backup / restore
- account settings and authentication

Core backend models:
- User
- Room
- Booking
- Client
- Rate
- Charge
- BookingType
- BookingCharge
- Payment
- ActivityLog
- HotelSetting
- RoomCleaningLog

8. WHAT THE SYSTEM MAINLY DOES

In practical terms, this system supports:
- front desk room availability monitoring
- reservation creation and editing
- check-in/check-out workflow tracking
- guest/client data storage
- room catalog management
- configurable discount and charge rules
- booking-type management
- payment recording and balance tracking
- financial and occupancy reporting
- printable statement of account output
- admin user administration
- hotel profile/logo maintenance
- backup and restore operations
- user profile, password, appearance, and 2FA settings

9. KNOWN STRUCTURAL NOTES

- The app uses Inertia pages instead of Blade views for most user-facing screens.
- Sidebar navigation is role-sensitive and reveals admin pages only for admin users.
- Date range selection influences dashboard, room, admin, and report pages.
- Booking and room status logic is deeply connected.
- Rates, charges, and booking types act as reusable configuration data for booking operations.
